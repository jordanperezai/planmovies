# Research

Dated research snapshots. Point-in-time deep dives. Written by /deep-research. Never updated after writing.

For living docs (specs, references, competitor tracking), use `docs/` instead. The split:
- `research/` = dated snapshots, never updated (like journal entries)
- `docs/` = living reference docs, updated in place (like HANDOFF.md)

If your project has brands (e.g. `brands/acme/`), each brand gets its own `docs/` and `research/` folders.

## Naming

All files are date-prefixed with a sequence number. Always include the number, even if it's the only entry that day.
```
YYYY-MM-DD-01-title.md           First entry that day
YYYY-MM-DD-02-title.md           Second entry that day
```

Always start with `01`. This prevents having to rename the first file when a second entry is added later.
