# Brainstorm — Ideas Without Commitment

Generate options. Name tradeoffs. Don't decide.

## When to use

- Too early to commit but need to see the landscape
- Multiple directions are plausible and you want them laid out
- Creative exploration before a council or decision
- When "just pick one" feels premature

## Step 1 — Define the space

What are we brainstorming about? State it in one sentence. "How should we handle auth?" not "auth."

## Step 2 — Generate 5-10 options

Each option gets:
- **One-line pitch** — what it is
- **Builds on** — what already exists (no blank-slate fantasies)
- **Tradeoff** — what you give up by choosing this
- **Effort** — trivial / light / medium / heavy

Don't filter. Don't rank. Don't eliminate. The point is breadth.

Include at least one option that feels too simple and one that feels too ambitious. The right answer is often between those.

## Step 3 — Name the dimensions

What are the 2-3 axes that matter? Speed vs thoroughness? Simple vs flexible? Cheap vs robust? Name them so the options can be compared on the same terms.

## Step 4 — Present and stop

Lay out the options. Name the dimensions. Stop.

Don't recommend. Don't rank. Don't say "option 3 seems best." The human picks. If they want a recommendation, they'll ask for a `/council`.

## Anti-Rationalization Table

| Excuse | Reality |
|--------|---------|
| "I should just pick one and start building" | Building the wrong thing is more expensive than 5 minutes of brainstorming. |
| "There are only two options" | There are always more than two. You haven't looked hard enough. Generate 5 minimum. |
| "I already brainstormed in my head" | Writing it down forces precision. "I thought about it" is not the same as "I laid out the tradeoffs." |
