# memory.md -- morph

> Sources: Zordon-OS migration (sessions 19-22, May 2026). Agent Collider morph (session 08, May 2026). Playback100 morph (2026-05-16). Terpmon morph (2026-05-16, 130+ session mature project). jordanperezai.com morph (2026-05-16, personal brand site).

## Validated Principles

- **Grep before you touch.** Before renaming or deleting ANY file, grep the entire repo (and sibling repos) for all references. One skill rename touched 5-15 files across Zordon-OS. `grep -rn "old-name" --include="*.md" --include="*.json" --include="*.ts" .` (last-confirmed: 2026-05-15)
- **Skill renames are the most expensive operation.** Each rename requires: mv directory, update SKILL.md name + triggers, update CLAUDE.md skills list, update every referencing skill, update ranger rosters, update business/product files. Batch all renames in one phase. (last-confirmed: 2026-05-15)
- **Commit rename + all references in the same commit.** A rename committed without reference updates breaks the repo for one commit. Caught in Zordon-OS LEARNINGS #9. (last-confirmed: 2026-05-15)
- **Ranger memory grep overcounts.** `grep -c "^\- \*\*"` counts ALL bold-prefixed bullets (dead directions, voice calibration, distillation log), not just validated principles. Always count manually within the Validated Principles section. This made 9 rangers appear over cap when only 3 were. (last-confirmed: 2026-05-15)
- **Declutter is mandatory post-migration, not optional.** New structure exposes old bloat. Ranger memories that looked fine pre-migration had 44 entries (cap: 20). The declutter got deferred 3 sessions on Zordon-OS. Run it as the final migration phase. (last-confirmed: 2026-05-15)
- **Content routing is concrete, not abstract.** When retiring files, route to specific destinations: platform gotchas → memory/topics/[platform].md, decision frameworks → ranger memory or skill memory, tool evaluations → research/, architecture details → MAP.md, voice content → VOICE.md. "Content flows UP" without a routing table is too vague. (last-confirmed: 2026-05-15)
- **Historical files keep old names.** Journal entries, session logs, and memory files are historical records. Don't update old references in them. Only fix references in files that are read for current operations. (last-confirmed: 2026-05-15)
- **File casing in Phase 0.** Root .md = CAPS. Subdirectory .md = lowercase-hyphens. Two exceptions: CLAUDE.md, SKILL.md. Only the root gets a README.md (project landing page). Enforce this early because later phases create new files that must follow the convention. (last-confirmed: 2026-05-25)
- **Dependencies between phases matter.** VOICE.md must exist before SOUL.md restructure (voice section moves out). Content must migrate before source file deletion. MAP.md absorbs ARCHITECTURE.md content before ARCHITECTURE.md is deleted. Document the dependency chain in the plan, not just the phases. (last-confirmed: 2026-05-15)
- **Sub-agents hallucinate file state.** When delegating migration work to parallel agents, they will fabricate directory structures and report files that don't exist. Give strict instructions: run ls/find first, only report what commands return. Verify key claims yourself. (last-confirmed: 2026-05-15)
- **Migration reveals naming problems.** When existing skills get reviewed against the scaffold, vague names become obvious. "ghl" means nothing to a stranger. "image-prompt" doesn't say the job. Budget a naming audit as part of migration. (last-confirmed: 2026-05-15)
- **Cross-repo sync is a phase, not an afterthought.** If the project has sibling repos using the same scaffold (Zordon-OS had Manfred + Zordon), stale references propagate. Check siblings after every rename/delete batch. (last-confirmed: 2026-05-15)
- **[Terpmon] Mature projects (130+ sessions) can't rename load-bearing files.** CONTEXT.md with 126 references across the codebase cannot be renamed or moved. Trim in place, extract content to memory/topics/, but keep the filename and section numbers. M3 pre-scan catches this: if refs > 50, the file stays where it is. (last-confirmed: 2026-05-16)
- **[Terpmon] Topic files are the decomposition mechanism for bloated gotcha files.** When a file is 500+ lines over cap (Terpmon CONTEXT.md: 867 lines), extract by theme to `memory/topics/{theme}.md`. Keep only the most recent entries in the source file + pointers to topic files. This is the knowledge graph pattern applied to monolithic reference files. (last-confirmed: 2026-05-16)
- **[Terpmon] Don't fight the auto-memory system.** Claude Code's auto-memory writes to MEMORY.md in its own format (one-line entries linking to topic files). The scaffold's hot/warm/cold structure is functionally equivalent. Keep whichever format the project already has. Forcing a format migration breaks the auto-memory tooling. (last-confirmed: 2026-05-16)
- **[Terpmon] CONTEXT.md decomposition is its own phase.** Budget it explicitly in the plan. On a 130-session project, CONTEXT.md was 868 lines and took more time than all other phases combined. Read the full file, identify theme clusters, create topic files, rewrite the source trimmed to under cap. (last-confirmed: 2026-05-16)
- **[Terpmon] Over-cap reference files get flagged, not rewritten during morph.** ARCHITECTURE.md at 858 lines is over the 500-line cap, but rewriting it during morph is diminishing returns. Create the structural gaps (VOICE.md, SECURITY.md, particle table), flag bloated files for /compress-docs, move on. Morph creates structure. Declutter handles bloat. Different jobs. (last-confirmed: 2026-05-16)
- **[Terpmon] The blocker hook is load-bearing infrastructure, not optional polish.** Every project needs destructive command blocking (rm -rf, git reset --hard, force push, DROP TABLE, curl|sh). Wire it in settings.json during the morph, not as a follow-up. It's in SECURITY.md as documentation AND in settings.json as enforcement. Both. (last-confirmed: 2026-05-16)
- **[Terpmon] Context guards grow organically, not scaffolded.** As a project discovers which file areas cause mistakes when edited without re-reading, it adds checks to a context-guard.sh hook. This replaces the old generic identity-guard.sh. Terpmon has 10 area-specific guards (pricing, auth, migrations, etc.). New projects start with none and grow them from real mistakes. (last-confirmed: 2026-05-16)
- **rangers/ vs personas/ requires reading the skill, not the name.** A directory called "owner-personas" inside rangers/ looked correct until you read the skill file and discovered it's a testing panel (focus group), not a debate ranger. The M2 audit must read the actual SKILL.md or session file to classify, not rely on directory names. Source: Manfred S08, Agent Collider morph. (last-confirmed: 2026-05-16)
- **Name rangers for what they DO, not what domain they cover.** Agent Collider's ranger went through three names: taxonomy → collider → strategy. "Taxonomy" described the domain. "Strategy" described the action. When stuck on a ranger name, ask: "what does this ranger do in a session?" Source: Manfred S08, Agent Collider morph. (last-confirmed: 2026-05-16)
- **Hooks belong in files, not inline JSON.** The scaffold Step 13 shows inline bash in settings.json for brevity. Real repos should extract each hook to hooks/[name].sh and point settings.json at the file. Inline bash is unreadable, untestable, and impossible to version meaningfully. Source: Manfred S08, ecosystem-wide hooks extraction. (last-confirmed: 2026-05-16)
- **Never present scaffold files as optional.** The agent's instinct is to ask "do you need this?" for files like WILL.md, MAP.md, hooks/. This wastes a round-trip and trains the human to expect gaps. Full morph means full. Adapt content, never skip files. Confirmed when Playback100 morph skipped 6 items then immediately needed all of them. (last-confirmed: 2026-05-16)
- **"Conditional Core" is a lie.** v6.3 listed WILL.md and infra/ as "Add If Applicable." Every project that ran more than 5 sessions needed both. Promoted to core in v6.4. If something was conditional and 3+ projects needed it, it's core. (last-confirmed: 2026-05-16)
- **Architecture belongs in MAP.md, not CLAUDE.md.** CLAUDE.md can have a brief architecture reference (5-6 bullets), but the full map (data flow, file ownership, dependency chains, component map, API dependencies) belongs in MAP.md. CLAUDE.md gets too long otherwise. (last-confirmed: 2026-05-16)
- **Infra scripts should fail gracefully.** auto-skill.sh checks WILL.md authorization. qmd-update.sh checks `command -v qmd`. message.sh checks .env exists. Scripts that hard-fail on missing dependencies prevent adoption. Copy them always, let them no-op until configured. (last-confirmed: 2026-05-16)
- **Related media folders consolidate naturally.** Three root folders (ai-portraits/, references/, inspiration/) that all serve the same system (portrait generation) should nest under one parent (portraits/). Grep blast radius was 15+ references across 6 files. Budget the reference updates when consolidating. (last-confirmed: 2026-05-16)
- **USER.md references the identity file, doesn't duplicate it.** When a project already has a deep identity file (JORDAN.md), USER.md should cover "how to work with this person" and point to the identity file for "who this person is." Don't duplicate content across both. (last-confirmed: 2026-05-16)
- **Flat-file skills get superseded by folder-based skills.** An old `wrap-up.md` flat file at `.claude/skills/wrap-up.md` coexists with the new `.claude/skills/wrap-up/SKILL.md`. The folder version takes precedence but the flat file still exists and causes confusion. Delete the flat file after copying the folder version. (last-confirmed: 2026-05-16)
- **Ship the full skill set. Don't cherry-pick.** First pass of jordanperezai.com morph created only 3 of 15 core skills. The user had to ask "why doesn't this run all the skills?" The instinct to skip "unnecessary" skills is the same failure as the "do you need this?" anti-pattern. Full morph = full skills. (last-confirmed: 2026-05-16)
- **Counts in the SKILL.md drift when the checklist grows.** When new files or skills are promoted to core, the prose references ("15 core files," "15 minimum") don't auto-update. After any version bump that changes the checklist, grep the file for stale counts. Caught in Zordon-OS S23: intro said 19, M2 said 15, checklist said 19. (last-confirmed: 2026-05-16)

## Dead Directions

- **Skill alias files (skill-aliases.json).** Tried creating a JSON mapping of old names to new names for backward compatibility. AI resolves via SKILL.md triggers field, not a lookup file. Created and deleted same session. (2026-05-15)
- **Premature directory creation.** Created research/personas/ with subdirectories before understanding what was needed. The real issue was the ranger NAME, not the data location. Deleted the whole structure. Build directories when you have content for them. (2026-05-15)
- **rm -rf on skill directories.** Hooks block it. Remove individual files with rm, then rmdir on the empty directory. (2026-05-15)
- **Grep-based ranger memory cap enforcement.** Cannot automate the count. The format is too varied (validated principles, dead directions, voice calibration entries all use `- **` prefix). Manual count within each section is required. (2026-05-15)
- **Asking "do you need this?" during morph.** Presenting a gap table with "Need it?" column caused the agent to skip WILL.md, MAP.md, TOOLS.md, hooks/, infra scripts, autonomous/, and multi-ranger. User had to ask "why didn't you do a full morph?" twice. The question itself is the failure. Full morph = no questions about scope. (2026-05-16)
