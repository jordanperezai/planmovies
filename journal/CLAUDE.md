# Journal

Narrative reflections on sessions. Not operational logs. Written in first person with voice, insight, and honesty about what worked, what didn't, and what shifted.

## Naming

```
YYYY-MM-DD-01-title.md           First entry that day
YYYY-MM-DD-02-title.md           Second entry that day
```

Always include the sequence number, even if it's the only entry that day. Start with `01`. This prevents having to rename the first file when a second entry is added later.

## Format

Every journal entry follows the magazine format:

```markdown
# [Title]

> Session N | YYYY-MM-DD | Topic tags
> Key discovery: "The one sentence you'd tell yourself yesterday."

---

## [Section Title]

[Narrative. Short paragraphs. Bold key moments.]

> [Pull quote for memorable lines or insights]

---

## What Shifted

[Closing insight. What's different now.]
```

**Rules:**
- Every entry needs a metadata line (session number, date, topic tags)
- Every entry needs a "Key discovery" quote (forces you to distill)
- Use `---` horizontal rules between distinct topics
- Use `>` blockquotes for pull quotes (memorable lines, key insights)
- Use `**bold**` for key moments within paragraphs
- Keep paragraphs short (3-4 sentences max)
- End with "What Shifted" section (what's different now vs when the session started)
- The bar: "would the human read this tomorrow and get value?"
