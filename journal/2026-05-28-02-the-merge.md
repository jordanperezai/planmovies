## The Build

The session started with a clear task. Mockup v4 had been approved last session. The job was to ship it. No exploration needed.

One node script, thirteen changes, 29/29 verification checks. That was the merge. The entire Movie tab — 227 lines of HTML, all its render functions, all its data — folded into the dashboard as three acts below the RSVP form. The old `page-briefing` section ceased to exist. The nav lost a tab. The evidence gained a timeline.

What surprised me was how clean the data survived the restructure. OFFICIAL_QUOTES, HYPE_HEADLINES, TRAILERS, BTS, RECOMMENDED — all 40+ quotes, 17 headlines, 27 content items — moved without losing a single entry. The node script pattern worked. Read the file once, make all changes in memory, write once. No file-sync errors.

## The Pipeline

After the merge, Jordan asked to run the full impeccable pipeline. We hadn't done that before in a single session. All five skills back to back.

/critique scored 26/40 heuristics, 3.5/10 AI slop. Up from 24/40 and 5.0/10 at the start of session 11. Three passes of work showing in the numbers.

The audits found things I hadn't caught: `--gold-dim` at 4.2:1 contrast (just below WCAG AA), radio inputs with `display: none` instead of sr-only positioning, inline style on zone-evidence killing its own side padding. Not glamorous fixes. Important ones.

The typography pass found 45 hardcoded font-sizes in a file with 174 tokenized ones. Most were exact matches (22px was `--text-xl`, 28px was `--text-2xl`). Four raw `font-family: 'Barlow Condensed'` declarations instead of `var(--display)`. The kind of drift that happens over twelve sessions of adding and editing. Fifteen minutes of node script fixed it.

## The Codex Review

The adversarial pass was the sharpest part of the session.

Codex found two P0 XSS vulnerabilities. Attendee names from Supabase going into innerHTML without `esc()` in some new code paths. Activity feed messages stored as HTML and rendered raw. Both real. Both fixed. Neither would have been caught by "it looks right."

The pattern from LEARNINGS #7 (onclick inline styles with user content) expanded to a broader rule: every function that builds innerHTML from Supabase data needs esc() on every user-derived field. We have the function. We just need the discipline to use it everywhere.

The other findings were structural. Double-bound CTA button. Keyboard radio navigation not syncing pill styling. Party size not resetting after RSVP Again. These are the bugs that don't show in normal testing. They show when someone uses the wrong input method, or when an edge case triggers the re-RSVP flow.

Eleven issues. All fixed. The file is cleaner for it.

## The Landing Page Moment

The last real design conversation of the session was about the poster carousel dots.

Jordan noticed them in the screenshot — three large gray circles below the poster. He said: if those were the users going, it would be more exciting. Seeing people you know before you even scroll.

The right call was to keep them in the same position. Swap dots for faces. Don't move anything else. The headcount text stays below the date. The social proof just becomes visual instead of abstract.

That's the Partiful lesson distilled. Not their bubbly pastel aesthetic. Their structural instinct: lead with people, not abstractions. The crew IS the conversion mechanism.

---

## What Shifted

The page now tells one story. It used to tell two half-stories on separate tabs. The evidence section made no sense without the film context. The film section made no case without the evidence. Now they share a scroll, separated by a single pivot line that does all the rhetorical work: "The movie is based on real events. These events are happening now."

That line has been in the design for two sessions. It only lands correctly when it's between the two things it connects.

The page was always one argument. We just finally put it on one scroll.
